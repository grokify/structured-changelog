# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html),
commits follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/),
and this changelog is generated by [Structured Changelog](https://github.com/grokify/structured-changelog).

## [0.6.0] - 2026-01-10

### Highlights

- TOON output format for LLM commands (~8x token reduction vs raw git log)
- GitLab repository support with automatic reference linking for issues, PRs, and commits
- Author attribution for external contributors and compact grouping of maintenance releases

### Added

- TOON (Token-Oriented Object Notation) as default output format for `parse-commits`, `suggest-category`, and `validate --format`
- `--format` flag for LLM commands supporting `toon` (default), `json`, and `json-compact`
- GitLab repository support for reference links and version comparisons
- Automatic reference linking: issues, PRs, and commits become clickable links when repository URL is provided
- `LinkReferences` renderer option (enabled in `FullOptions`) for hyperlinking references
- `versioning` field in changelog IR: `semver`, `calver`, `custom`, or `none`
- `commit_convention` field in changelog IR: `conventional` or `none`
- Dynamic header generation based on versioning scheme and commit convention
- Validation error codes E008 (invalid versioning) and E009 (invalid commit convention)
- `maintainers` and `bots` fields in changelog IR for identifying team members
- Auto-detection of common bots (dependabot, renovate, github-actions, etc.)
- `IsTeamMember()` method for checking if author is maintainer or bot
- `IncludeAuthors` renderer option for external contributor attribution
- Automatic author attribution with linked GitHub/GitLab profiles
- Inline attribution stripping to avoid duplicate attribution when author field matches
- `CompactMaintenanceReleases` renderer option for grouping maintenance-only releases
- `IsMaintenanceOnly()` method to detect dependency/documentation-only releases
- `--changelog` flag for `parse-commits` to auto-detect external contributors
- `IsExternal` field in commit output marking external contributors
- `Contributors` summary in `parse-commits` output with deduplicated author list
- `IsTeamMemberByNameAndEmail()` for matching by git author name, email, or GitHub noreply username

### Changed

- **BREAKING:** LLM commands now output TOON format by default for ~40% token reduction vs JSON
- **BREAKING:** `parse-commits` and `suggest-category`: `--compact` flag replaced with `--format` flag
- **BREAKING:** `validate`: `--json` flag replaced with `--format` flag
- Commit references displayed as short hash (7 chars) with backticks in linked output
- Consecutive maintenance releases grouped into compact sections by default

### Documentation

- TOON integration design document
- Updated LLM guide with TOON examples and ~8x token efficiency
- Reference linking documentation in specification
- Claude Code project configuration with `/changelog` slash command

## [0.5.0] - 2026-01-04

### Highlights

- LLM-assisted changelog generation with optimized git parsing (~5x token reduction) and automatic commit classification

### Added

- `parse-commits` command for LLM-optimized git history output (~5x token reduction vs raw git log)
- `suggest-category` command for commit classification with confidence levels
- `--json` flag for `validate` command with structured error output, line numbers, and fix suggestions
- MkDocs documentation site with Material theme
- LLM integration guide with prompts and workflows
- PRD documents for planned localization and rendering features

### Documentation

- Release notes consolidated in `docsrc/releases/` for MkDocs site

## [0.4.0] - 2026-01-04

### Highlights

- Extend Keep a Changelog with 13 additional change types organized into tiers for flexible filtering and validation

### Added

- Tier-based change type system with 4 levels: **core** (KACL standard), **standard** (common extensions), **extended** (docs/build/acknowledgments), **optional** (operations/internal)
- 19 change types in canonical order (see `CHANGE_TYPES.json`)
- New change types beyond Keep a Changelog: Highlights, Breaking, Upgrade Guide, Performance, Dependencies, Documentation, Build, Infrastructure, Observability, Compliance, Internal, Known Issues, Contributors
- CLI flags: `--max-tier` for generate, `--min-tier` for validate
- Library functions: `Summary()`, `ValidateMinTier()`, `ParseTier()`, `OptionsFromConfig()`
- Canonical ordering documentation in README.md

### Changed

- Change types now support tier-based filtering for validation and generation
- CLI refactored to use library functions for better testability

### Tests

- Comprehensive unit tests achieving 98%+ library coverage (changelog: 98.2%, renderer: 98.9%)

## [0.2.0] - 2026-01-03

### Highlights

- Generated changelogs now include clickable version comparison links for easy navigation on GitHub

### Added

- GitHub version reference links at bottom of rendered Markdown
- Auto-detect GitHub repository from `repository` field
- First release links to `/releases/tag/vX.Y.Z`
- Subsequent releases link to `/compare/vA...vB`
- Unreleased section links to `/compare/vLatest...HEAD`
- Dependabot configuration for Go modules and GitHub Actions

### Changed

- Add Structured Changelog attribution to rendered Markdown header

## [0.1.0] - 2026-01-03

### Highlights

- Machine-readable JSON changelog with deterministic Markdown generation following Keep a Changelog format

### Added

- JSON IR schema for structured changelogs (v1.0)
- `changelog` package with IR types, parsing, and validation
- `renderer` package with deterministic Markdown generation
- `sclog` CLI with `validate` and `generate` subcommands
- Support for Keep a Changelog categories: added, changed, deprecated, removed, fixed, security
- Optional security metadata: CVE, GHSA, severity, CVSS, CWE
- Optional SBOM metadata: component, version, license
- JSON Schema for IR validation
- Example changelogs: basic, security, full
- Documentation: spec, security guide, SBOM guide
