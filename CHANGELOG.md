# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html),
and this changelog is generated by [Structured Changelog](https://github.com/grokify/structured-changelog).

## [0.5.0] - 2026-01-04

### Added

- `parse-commits` command for LLM-optimized git history output (~5x token reduction vs raw git log)
- `suggest-category` command for commit classification with confidence levels
- `--json` flag for `validate` command with structured error output, line numbers, and fix suggestions
- MkDocs documentation site with Material theme
- LLM integration guide with prompts and workflows
- PRD documents for planned localization and rendering features

### Documentation

- Release notes consolidated in `docsrc/releases/` for MkDocs site

## [0.4.0] - 2026-01-04

### Added

- Tier-based change type system with 4 levels: **core** (KACL standard), **standard** (common extensions), **extended** (docs/build/acknowledgments), **optional** (operations/internal)
- 19 change types in canonical order (see `CHANGE_TYPES.json`)
- New change types beyond Keep a Changelog: Highlights, Breaking, Upgrade Guide, Performance, Dependencies, Documentation, Build, Infrastructure, Observability, Compliance, Internal, Known Issues, Contributors
- CLI flags: `--max-tier` for generate, `--min-tier` for validate
- Library functions: `Summary()`, `ValidateMinTier()`, `ParseTier()`, `OptionsFromConfig()`
- Canonical ordering documentation in README.md

### Changed

- Change types now support tier-based filtering for validation and generation
- CLI refactored to use library functions for better testability

### Tests

- Comprehensive unit tests achieving 98%+ library coverage (changelog: 98.2%, renderer: 98.9%)

## [0.2.0] - 2026-01-03

### Added

- GitHub version reference links at bottom of rendered Markdown
- Auto-detect GitHub repository from `repository` field
- First release links to `/releases/tag/vX.Y.Z`
- Subsequent releases link to `/compare/vA...vB`
- Unreleased section links to `/compare/vLatest...HEAD`
- Dependabot configuration for Go modules and GitHub Actions

### Changed

- Add Structured Changelog attribution to rendered Markdown header

## [0.1.0] - 2026-01-03

### Added

- JSON IR schema for structured changelogs (v1.0)
- `changelog` package with IR types, parsing, and validation
- `renderer` package with deterministic Markdown generation
- `sclog` CLI with `validate` and `generate` subcommands
- Support for Keep a Changelog categories: added, changed, deprecated, removed, fixed, security
- Optional security metadata: CVE, GHSA, severity, CVSS, CWE
- Optional SBOM metadata: component, version, license
- JSON Schema for IR validation
- Example changelogs: basic, security, full
- Documentation: spec, security guide, SBOM guide
