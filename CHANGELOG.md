# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html),
and this changelog is generated by [Structured Changelog](https://github.com/grokify/structured-changelog).

## [0.3.0] - 2026-01-04

### Added

- GoReleaser configuration for multi-platform binary releases (Linux, macOS, Windows Ã— amd64, arm64)
- GitHub Actions release workflow triggered by version tags
- Homebrew tap distribution via `brew install structured-changelog`
- Binary symlink: both `sclog` and `structured-changelog` commands available

## [0.2.0] - 2026-01-03

### Added

- GitHub version reference links at bottom of rendered Markdown
- Auto-detect GitHub repository from `repository` field
- First release links to `/releases/tag/vX.Y.Z`
- Subsequent releases link to `/compare/vA...vB`
- Unreleased section links to `/compare/vLatest...HEAD`
- Dependabot configuration for Go modules and GitHub Actions

### Changed

- Add Structured Changelog attribution to rendered Markdown header

## [0.1.0] - 2026-01-03

### Added

- JSON IR schema for structured changelogs (v1.0)
- `changelog` package with IR types, parsing, and validation
- `renderer` package with deterministic Markdown generation
- `sclog` CLI with `validate` and `generate` subcommands
- Support for Keep a Changelog categories: added, changed, deprecated, removed, fixed, security
- Optional security metadata: CVE, GHSA, severity, CVSS, CWE
- Optional SBOM metadata: component, version, license
- JSON Schema for IR validation
- Example changelogs: basic, security, full
- Documentation: spec, security guide, SBOM guide
