## Changelog Management <a href="#task-list">↑ Top</a>

<a id="incremental-update"></a>

### [ ] Incremental Update Command

Add new releases to existing CHANGELOG.json without regenerating everything

- [ ] schangelog add-release CHANGELOG.json --version=vX.Y.Z --since=vPREV
- [ ] schangelog update CHANGELOG.json --since=LATEST (auto-detect latest version)
- [ ] Preserve existing entries, prepend new release

<a id="auto-highlights"></a>

### [ ] Auto-Suggest Highlights

Suggest highlights for releases based on commit analysis

- [ ] schangelog suggest-highlights CHANGELOG.json --version=vX.Y.Z
- [ ] Prioritize feat commits, breaking changes, and large diffs
- [ ] Output suggested highlight text for manual review

<a id="merge-command"></a>

### [x] Merge Command

Merge new release entries into existing CHANGELOG.json or combine multiple changelog files

- [x] schangelog merge base.json additions.json -o output.json
- [x] schangelog merge CHANGELOG.json --release release.json (prepend single release)
- [x] Preserve metadata from base file (maintainers, bots, repository)
- [x] Detect and warn about duplicate versions
